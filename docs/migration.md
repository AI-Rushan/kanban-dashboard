# Перенос на другой компьютер — «Канбан-дашборд»

Версия: 1.0 | Дата: 2026-06-11

---

## Суть

Данные дашборда хранятся в браузере (localStorage). Они привязаны к конкретному браузеру на конкретном компьютере. При переезде на новый компьютер или в новый браузер данные нужно перенести через JSON-файл.

Весь процесс занимает 3–5 минут.

---

## Экспорт данных (на старом компьютере)

1. Открой дашборд
2. Нажми кнопку **📤** в тулбаре
3. Скачается файл `kanban_backup_ДАТА.json`
4. Перекинь файл на новый компьютер (через AirDrop, флешку, облако, почту)

---

## Установка на новом компьютере

Скопируй в папку `kanban` (например, `~/Desktop/kanban/`) три файла:
- `dashboard.html`
- `archive.html`
- `launch.command`

Если нужен Google Calendar — выполни шаги из `installation.md`. Для базового использования достаточно просто открыть `dashboard.html`.

---

## Импорт данных (на новом компьютере)

1. Открой дашборд
2. Нажми кнопку **📥** в тулбаре
3. Выбери файл `kanban_backup_ДАТА.json`
4. Данные появятся сразу — перезагрузка не нужна

---

## Особый случай: переезд с file:// на localhost

Если ты раньше открывал дашборд двойным кликом (`file://`), а теперь переходишь на `launch.command` (`http://localhost:8080`) — данные в браузере НЕ переедут автоматически. Это разные «адреса» с разным хранилищем.

**Решение:**
1. Открой дашборд через `file://` (двойной клик на `dashboard.html`)
2. Сделай экспорт 📤
3. Закрой вкладку
4. Запусти через `launch.command`
5. Сделай импорт 📥 из только что сохранённого файла

---

## Передача дочери

1. Скопируй папку `kanban` на флешку или отправь через AirDrop
2. На её компьютере: выполни `chmod +x launch.command` в Терминале (если нужен Calendar)
3. Данные к ней не переедут — она начнёт с чистого листа
4. Если нужно передать задачи — сделай экспорт и отправь ей JSON-файл для импорта

---

## Google Calendar на новом компьютере

Google Calendar настройка хранится в localStorage. Значит, на новом компьютере её нужно настроить заново:

1. Повтори шаги из `installation.md` раздел «Настройка Google Calendar»
2. Client ID — тот же самый (из Google Cloud Console)
3. Добавь новый email в Test users, если он другой
4. В дашборде заново нажми 📅 → вставь Client ID → подключись → выбери календарь

> Задачи, у которых был `gcalEventId`, после импорта JSON снова привяжутся к Calendar при первой синхронизации — если события там ещё существуют.

---

## Рекомендации по хранению бэкапов

- Делай экспорт **раз в неделю** (кнопка 📤, 5 секунд)
- Храни последние 3–4 файла — если что-то пошло не так, можно откатиться
- Удобные места: iCloud Drive, папка «Документы», почта себе
- Имя файла содержит дату — старые можно удалять когда их уже 5+
