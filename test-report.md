## Детали по пройденным сценариям

### TS-01 · completedAt — правильная дата ✅

Проверено для **критического случая: 01:30 ночи по Москве** (UTC = 22:30 предыдущего дня).

| Проверка | Результат |
|----------|-----------|
| `todayStr()` в 01:30 МСК → `2026-06-14` | ✅ |
| `todayStr()` в 12:00 МСК → `2026-06-14` | ✅ |
| `moveCard` → Done: `completedAt = 2026-06-14` при 01:30 МСК | ✅ |
| `moveCard` → из Done: `completedAt = null` | ✅ |

До исправления старый код возвращал `2026-06-13` (UTC-дата) — баг устранён.

### TS-03 · Длительность события в Calendar ✅

`gcalAddDay()` теперь использует локальные компоненты даты.

| Входное значение | Результат |
|-----------------|-----------|
| `2026-06-14` | `2026-06-15` ✅ |
| `2026-06-30` (конец месяца) | `2026-07-01` ✅ |
| `2026-12-31` (конец года) | `2027-01-01` ✅ |

До исправления: `gcalAddDay('2026-06-14')` → `2026-06-14` (создавало нулевую длительность события).

---

### TS-10 · Повторная синхронизация не меняет дедлайн ✅ (анализ кода)

После каждого PATCH сервер возвращает `updated`, и код записывает:
```js
if(patched?.updated) task.lastModified = new Date(patched.updated).getTime();
```
При повторной синхронизации `taskTs === eventTs` → `taskTs >= eventTs` → PATCH снова отправляется с теми же данными — данные не теряются.

## 🔍 Обнаруженный баг (не из списка исправлений)

### BUG-01 · Статус «🔥 сегодня» не отображается для всесуточных задач

**Серьёзность:** 🟡 Важный  
**Обнаружен:** в ходе тестирования `dateStatus()`

**Суть:** Для всесуточной задачи с дедлайном сегодня вместо `🔥 сегодня` показывается `⏰ через 1д`.

**Причина:**
```js
function deadlineDt(dl, dlTime){
  return dlTime ? new Date(dl+'T'+dlTime+':00')
                : new Date(dl+'T23:59:59');  // ← конец дня
}
// ref = TODAY = 00:00:00 (начало дня)
// diffMs = 23ч 59м 59с → diffDays = Math.ceil(0.9999) = 1 → 'soon'
```
`diffDays === 0` (условие для `'today'`) недостижим для всесуточных задач — дедлайн всегда конец дня, а `ref` — начало.

**Ожидаемое поведение:** задача с дедлайном сегодня должна показывать `🔥 сегодня`.  
**Текущее поведение:** показывает `⏰ через 1д`.

**Предлагаемое исправление:**
```js
function dateStatus(dl, compareTo, dlTime){
  ...
  // Для всесуточных задач: 'today' если deadline = сегодня
  if(!dlTime && !compareTo){
    const dlStr = dl; // 'YYYY-MM-DD'
    const todayDate = todayStr();
    if(dlStr === todayDate) return 'today';
  }
  ...
}
```
